信息安全三要素 CIA Triad

  1. 保密性（Confidentiality）：信息在传输时不被泄露
  2. 完整性（Integrity）：信息在传输时不被篡改
  3. 有效性（Availability）：信息的使用者是合法的

公钥私钥解决保密性问题。
数字签名解决完整性问题和有效性问题。

加密

对称

  • 加密、解密使用一个共同的密钥。
  • 特点: 运算成本较低，需要保证密钥不泄漏，管理麻烦。
  • 常见算法: DES AES

非对称(公钥)

  • 成对的两个密钥，选取一个作为公钥，另一个作为私钥
    • 一个密钥所加密的内容只能由另一个密钥来解密
    • 通信双方都要各自维护一对密钥(给予对方自己的公钥)
  • 特点: 更安全，运算慢
  • 常见算法: RSA

    RSA 基本原理

    一文搞懂 RSA 算法
  • 密钥生成
    • n = p * q (p,q都为质数)
    • 欧拉函数φ(n)=(p-1)(q-1)， 记结果为m
    • 随机取一个整数e，与m互质(公约数只有1)
    • 找一个整数 d，使得 (e * d ) % m = 1
    • 得到密钥对: (n,e) (n,d)
  • 加解密
    • 加密 src^e % n = ret
    • 解密 src^d % n = ret
  • 有没有可能在已知 （n，e) 的情况下，推导出 d
    • 想知道 d 需要知道欧拉函数 m = φ(n)(小于等于 n ，与 n 互质的数的个数)
    • 想知道 m 需要知道 P 和 Q
    • 要知道 P 和 Q 需要对 n 进行因数分解

Hash算法(摘要算法)

  • 不可逆，运算快，信息验证
  • MD5 SHA

数字签名和数字证书

主要目的是认证而非加密？确保数据内容可靠。
防止信息被破坏(通俗的说，就是我得不到的信息，你也别想得到！)
如果识别出文件被破坏，则直接丢弃

数字签名

  • 目标问题： 证明文件未被篡改
    • 前提: 不对大文件进行高复杂的加密(如非对称)
  • 方法： 对文件的摘要信息(hash)进行加密(用自己的私钥)
    • 然后内容可以选择用对方的公钥或随机生成的对称密钥加密
  • 接收方使用对方的公钥进行解密，并比对hash值，从而证明数据来源，和是否被篡改

    数字证书

  • 目标问题： 验证公钥
    • 公钥可能被截取，从而获得了的是被伪造了的公钥
  • 认证机构（Certification Authority，CA）
    • 受信任的第三方机构
  • 流程
    • 将公钥 A 给 CA
    • CA用私钥对公钥 A，生成数字签名，整合必要信息，构成证书
    • 将证书和公钥 A一起给对方，对方在进行校验(一般操作系统自带或者后续安装相关的CA的公钥)

Https

  • HTTPS在传输的过程中会涉及到三个密钥：
    • 服务器端的公钥和私钥，用来进行非对称加密
    • 客户端生成的随机密钥，用来进行对称加密
