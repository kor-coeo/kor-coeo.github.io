SVN 与 Git

发表于 2019-08-26 | 更新于 2019-09-12 | 分类于 软件工程 | 评论数：

svn

工具组

服务端相关

  • svnadmin: 创建、调整、修复 仓库 （文件创建）
  • 服务方式:
    • svnserve: SSH
    • mod_dav_svn: Apache HTTP

管理相关

  • svnlook
  • svnversion
  • svnsync
  • svnrdump

客户端

  • svn

创建库

svnadmin create /home/.../DIR
新建的仓库中会包含不少初始文件，如conf文件夹内会包含不少配置文件。

conf 文件夹

svnserve.conf

  • anon-access: 未验证用户权限，”write”、”read”和”none”，write为可读可写，read只读，none即为禁用。
  • auth-access:
  • authz-db: 权限配置文件所在。可为对conf的相对路径。conf目录下已有一个默认的 authz 文件。

passwd

密码文件，是简单明文？格式：<用户名> = <口令>

authz

权限配置（分组）

创建服务

svnserve -d -r /home/.../DIR
-d: daemon,作为守护进程，后台执行。-r + DIR 指定根目录。

远程仓库

配置账户信息

  • git config –global user.name “kor-coeo”

SSH keys

  • ssh-keygen -t rsa -b 4096 -C “kor.coeo@gmail.com“

  • cat ~/.ssh/id_rsa.pub
