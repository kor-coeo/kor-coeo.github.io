Tips

  • 云项目
  • 高并发 goroutine
  • 简化编程语言相比于在臃肿的语言上不断增加新特性，会是更大的进步。
  • 速度整体上优于Java，在像C方向努力
  • NextC，面向过程的语言
  • 快速编译

目录结构

src

  • 文件夹 对应 代码包
  • 根目录 为 main 包

pkg

  • 归档，包编译完后的 .a 文件

bin

  • 可执行文件

源码 .go

命令源码

  • main包(src/),且含main方法，可以直接 go run + file.go 执行
  • 多个命令源码可以分别 run 执行，但是项目 build 要求只有一个命令源码

库源码

  • 包名 与 文件夹名 一致，且不包含 main

测试源码

  • 文件名以 _test.go 结尾
  • 需要包含一个函数：
    • 类型为： testing.T 或 testing.B
    • 名称开头为： Test 开头或 Benchmark
  • go test 执行当前目录（包）下的所有 test 文件

包

  • package
    • 工具包内也可声明为main包

  • import

    • 别名：
      • . : 相当于namespace
      • _ : 引用而不使用，初始化，意义不明

  • 包初始化 func init(){}

    • 在 main 执行前，执行且只执行一次

基本语法

语句结尾无需 “;”

基本类型

  • 布尔 bool
  • 字符串 string
  • 整型 int int32(rune)
  • 无符号 uint uint(byte) uintptr
  • 浮点 float32 float64
  • 复数 complex64 complex128

变量声明：

var 变量名 变量类型
自动类型 只能在函数内部使用：
名字 := 表达式
a,s:=1, "abc"
直接初始化也可以：
var attack = 40
变量交换：
b, a = a, b
_ 匿名变量：垃圾桶，函数 返回多值 时 占位
