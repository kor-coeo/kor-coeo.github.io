    背景

  • 协议 x(x11) –> 服务器 Xorg –> 客户端 Gnome
  • OpenGL 协议 –> GLFW 实现
  • Cmake –> 自动生成 makefile 的工具，然后 make
  • glad 一个 OpenGL 相关的，在线且便捷的项目初始化工具
    • 自动引入头文件、OpenGL对象指针管理
  • 关于 makefile：
    • CFLAGS : -c –> .o(中间文件,不连接); -o –> out: 连接成可执行文件; -I – include dir; -g –> G-DB debug
    • LDFLAGS : -l/-L –> lib: .a .so / dir; –static –> 静态，生成自带依赖库的文件
    • 参数顺序是有要求的！！！！！！！！！！！

环境配置

GLFW

  • 下载的源码包里有2个东西要用：
    1. include文件夹（ 直接用 就可以）
    2. 编译后得到的库文件: 根目录下使用cmake生成makefile，然后编译到指定目录，目标目录内的src文件夹里就是要用的库文件了(libglfw3.a)

-lGL

sudo ln -s libGL.so.1 /usr/lib/libGL.so

Hello Window

  • 基本步骤
    1. glfw 的初始化
    2. 通过 glfw 创建窗口对象
    3. 初始化 glad
    4. 像 GL 申请画布：glViewport
  • 注册回调函数
    • GLFW 提供形如 glfw…Callback( window, my_callback) 的方法

Hello Triangle

Graphics pipeline 渲染管道

1.顶点 -> 2.连线 -> 3.图形（可选，一般无操作） -> 4.光栅化（获得片段，一个片段就是一个像素）-> 5.片段着色 -> 6.测试（alpha test，透明处理）与混合（blend）

创建顶点

  • x，y，z 均应在[-1,1],否则丢弃
  • vertex buffer objects (VBO)：在 显存 中储存顶点
    • 在OpenGL中具体为： GL_ARRAY_BUFFER

编写 顶点Shader

  • 专用语言：GLSL (OpenGL Shading Language)
  • layout (location = 0)
  • 上面视为字符串，下面是在C源码中调用
  • 可在 运行时 动态 编译
  • GL_VERTEX_SHADER

Fragment（光栅） Shader

RGB_A

  • FragColor = vec4(1.0f, 0.5f, 0.2f, 1.0f);
  • GL_FRAGMENT_SHADER

连接 shader

  • glCreateProgram();

顶点

  • X，Y,Z
  • 32bit(4字节)浮点 x3
  • VBO是一个顶点也是数组，一个独立图形的？
  • Vertex Array Object, VAO - 顶点数组。数组的数组。
    • GL_ARRAY_BUFFER
    • glDrawArrays
  • 索引缓冲对象(Element Buffer Object，EBO，也叫Index Buffer Object，IBO)。 储存顶点的索引
    • GL_ELEMENT_ARRAY_BUFFER
    • glDrawElements
