Spring 的 特质

  • IOC(DI)
    • 控制反转，实现方式是依赖注入
    • 主动创建、配置实例化对象，到由容器自动分配
    • 可以自动处理对象间的依赖
    • 反转？大分，理解成托管比较好。
    • IOC容器负责 创建对象 和 配置成员属性(通过Bean xml)
  • AOP Aspect Oriented Programming

Spring 做了些什么？

创建和配置对象属性

  1. 编写 类

  2. 编写 Bean XML(实例对象描述)

  3. 获得 IOC容器 对象 (Context)
    ApplicationContext ctx = new ...("xxx.xml")

  4. 获得Bean实例对象
    ClassPoi instance = (ClassPoi) ctx.getBean("index-id")

Bean配置

基本概念

  • 实现方式：Java的反射机制(需要一个无参构造器)
  • IOC容器：
    • BeanFactory：底层，面向Spring框架本身
    • ApplicationContext：面向使用Spring框架的开发者
  • getBean：
    • id
    • 类-类型，要求 有且只有 实例化对象

依赖注入 & bean.xml的编写

  1. 属性注入(常用)
  2. 构造器注入
    • 可选用 index 指定参数顺序
  3. 数据类型：
    • 对象(非基本数据类型)用: ref = "instance-id"
    • 特殊字符: <value><![CDATA[...]]></value>
    • value Of 集合： list set map(entry key value) props(类似级联)
    • util:list 在外部定义可复用的list
  4. 级联赋值：
    1. ref = obj
    2. property name=obj.xxx value= xxx
  5. 自动装配 @autowire
    • byName or byType
    • 自己开发可以少用，保证文档清晰。整合第三方模块，才更多地会需要自动装配类。
  6. xml可以通过继承、依赖进行复用(针对xml文件内)
    • abstract=”true” 抽象Bean(可以没有class属性，只有成员属性模板)
    • parent=”beanTeamplate”
    • 依赖 depends-on(指示IOC容器先去创建好依赖项)
  7. 作用域 单例(初始化context时即创建) or 原型(需要时才创建)
    • （service模块等）向IOC容器申请依赖项时的分配方式

引用外部配置文件 .properties

  • <context:property-placeholder location="classpath:xxx.properties"/>
  • value="${props-name}"

SpEL SP-表达式-语言

动态赋值

  • #{…} #{‘…’}
    • 替代 ref: value="#{bean-id}"
    • 链式操作： value="#{obj.funA().funB()}"
    • 支持运算
      • + - * / % ^
      • 字符串拼接
      • 比较
      • …? x:y
    • 静态类 T():value="#T(java.lang.Math).PI"
