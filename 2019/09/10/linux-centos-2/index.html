    Openssl 自签证书

  • 构建证书

    • mkdir -pv /etc/pki/CA/{certs,crl,newcerts,private} && touch /etc/pki/CA/{serial,index.txt} && echo 01 >> /etc/pki/CA/serial
    • openssl genrsa -out /etc/pki/CA/private/cakey.pem 2048

    • vim /etc/pki/openssl.cnf

    • openssl req -new -x509 -key /etc/pki/CA/private/cakey.pem -out /etc/pki/CA/cacert.pem -extensions v3_req -config /etc/pki/openssl.cnf

  • 颁发证书
    • openssl genrsa -out httpd.key 2048生成密钥,二之一(.key)
    • openssl req -new -key httpd.key -out httpd.csr -extensions v3_req -config /etc/pki/openssl.cnf
    • openssl ca -in httpd.csr -out /etc/pki/CA/certs/httpd.crt二之一(.crt),颁发证书(CA服务器做的事，由于这里是自签的形式，https服务器是同一台机器)
      • docker容器内出现错误，修改配置文件中的默认路径: vim /opt/gitlab/embedded/ssl/openssl.cnf

端口绑定

  • gitlab-ctl reconfigure

  • vim /etc/gitlab/gitlab.rb

  • By default NGINX will listen on the port specified in external_url or implicitly use the right port (80 for HTTP, 443 for HTTPS). If you are running GitLab behind a reverse proxy, you may want to override the listen port to something else. For example, to use port 8081:nginx['listen_port'] = 8081

    • 如果在端口映射处(如Ngnix反向代理)做的ssl，而内部采用的是http，需要禁用gitlab自带的nginx['listen_https'] = false
  • gitlab_rails['gitlab_shell_ssh_port'] = 233改变web界面显示的git(ssh)端口，似乎内部实现还是22端口?
