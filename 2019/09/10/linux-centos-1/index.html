CentOS 配置

发表于 2019-09-10 | 更新于 2019-09-26 | 分类于 Ops | 评论数：

用户管理

useradd {name-poi} -U -m

  • m: create home
  • U: create the same name group
  • G: addition group

userdel -rf

passwd {name-poi}

visudo

分配 sudo 权限

Group

  • groupadd admin
  • usermod -G 添加附加组
  • newgrp g-name 更新用户组(手动编辑文件的话)

查看 cat

  • /etc/passwd
  • /etc/group
    • 这里可以将用户从组中移除

防火墙 firewall

  • firewall-cmd --state
    • 检查是否正在运行

开启、关闭

  • systemctl start/stop/restart firewalld
  • systemctl enable/disable firewalld 开机启动

应急模式（应急模式阻断所有网络连接，防止出现紧急状况）

  • firewall-cmd --query-panic
  • firewall-cmd --panic-on/off

端口配置

  • 检查
    • firewall-cmd --query-port=80/tcp
    • firewall-cmd --zone=public --list-ports
    • firewall-cmd --list-services
    • firewall-cmd --list-all
  • 配置
    • firewall-cmd --add-port=80/tcp --permanent –permanent重启保留
    • firewall-cmd --remove-port=80/tcp
    • firewall-cmd --permanent --add-rich-rule="rule family="ipv4" source address="192.168.11.0/24" port protocol="tcp" port="6689" accept"这种需要firewall-cmd --reload
    • 移除规则vim /etc/firewalld/zones/public.xml

ssh

查看登录记录

  • who /var/log/wtmp
  • more /var/log/secure

修改端口

  1. vim /etc/ssh/sshd_config

  2. 重启

    • systemctl restart sshd.service
    • systemctl status sshd.service查看信息:error: Bind to port 233 on 0.0.0.0 failed: Permission denied
    • 需要临时关闭SELinux

SELinux

  • getenforce查看是否启用
  • setenforce 0临时关闭
  • setenforce 1启用

iptables

未使用，先略

  • iptables -A INPUT -s 192.168.99.0/24 -p tcp --dport 22 -j ACCEPT
  • 永久生效
    • service iptables save
    • cat /etc/sysconfig/iptables

RSA

  1. vim .ssh/authorized_keys
    • 放入 id_rsa.pub
  2. vim /etc/ssh/sshd_config
    • RSAAuthentication yes
    • PubkeyAuthentication yes
  3. 重启ssh服务
    • setenforce 0
    • service sshd restart
    • setenforce 1

防止自动断开

编辑客户端(ubuntu): /etc/ssh/ssh_config

  • ServerAliveInterval 50
  • ServerAliveCountMax 3
    • sudo service sshd restart
    • 注意，非sshd_config(服务器配置)

Docker

安装

  • Remove old version
  • Install using the repository
  • 检验

更新CentOS

Mysql

删除自带的 mariadb

  • rpm -qa|grep mariadb (mysql)
  • rpm -e --nodeps ...

安装 Mysql5.7

配置yum源

  • MySQL Yum Repository
    • https://dev.mysql.com/get/mysql80-community-release-el7-3.noarch.rpm
  • wget ...
  • 安装源sudo rpm -ivh ...
  • 选择安装5.7版本
    • yum repolist all | grep mysql
    • yum-config-manager --disable mysql80-community
    • yum-config-manager --enable mysql57-community

安装Mysql

  • sudo yum install mysql-community-server
  • systemctl status mysqld.service
  • systemctl start mysqld.service

密码配置

  • 新版本的Mysql会自动创建一个初始密码
    • sudo grep 'temporary password' /var/log/mysqld.log
  • 修改密码，mysql里面:
    • set password for 'root'@'localhost' = password('...');
